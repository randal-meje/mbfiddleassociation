**Known data gap:** 2015 likely has a 4th inductee (Cecil Isliefson) whose primary
slideshow caption doesn't contain a category keyword, so the parser skipped him.
See `migration/output/wall-of-fame/2015.json` → `photos[]`; the board should
review and manually add him.

**Schema implication for Sanity:** the `inductee` schema field `bioPdfEn`
should be optional, not required. The `photo` field should be promotable
to "primary" (one of many). `bioBody` (rich text) is the best long-term home for
the 2013–2019 biographical content once it's written up — the PDFs for 2024 can
coexist with those, or be superseded over time.

7, Other 1.

## Technical Notes / Limitations

- Scraped with polite `User-Agent: MFA-Migration-Scraper/1.0` and a 250 ms
  delay between requests.
- Assets are stored under `output/assets/uploads/...` mirroring Weebly's
  folder structure. The `assets/index.json` file maps original URL → local
  path → referencing page URLs.
- The scraper is idempotent on assets (it skips already-downloaded files), so
  re-running is cheap. It is **not** idempotent on JSON output — re-running
  overwrites everything under `output/`.
- `events.html` was intentionally skipped (Google Calendar iframe; events will
  be handled by a new internal subsystem).
